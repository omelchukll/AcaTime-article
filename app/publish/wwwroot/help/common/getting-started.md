# Getting Started with AcaTime

AcaTime follows a logical data-entry workflow. The application menu is organised in the recommended order of operations.

## Workflow

### 1. Administration

Before anything else:
- Create faculties (system administrator only)
- Add users and assign roles

### 2. General Setup

Fill in the reference dictionaries:
- **Academic Semester** — create a new semester with start and end dates
- **Year of Study** — add study years (1st year, 2nd year, etc.)
- **Subject Types** — create subject types (lecture, practical, laboratory, etc.)
- **Subgroup Types** — configure how groups are split into subgroups
- **Classroom Types** — add classroom types (general, computer lab, etc.)

### 3. Academic Process

After the reference data is ready:
- **Classrooms** — enter available rooms and assign types
- **Educational Programs** — create educational programs for the faculty
- **Groups** — add student groups and define subgroups
- **Study Disciplines** — create disciplines for educational programs
- **Teachers** — add teachers and their workload
- **Group Subjects** — assign specific classes to groups and subgroups

### 4. Schedule

Once all data is in place:
- **Constraints & Estimation** — configure rules and constraints for schedule generation
- **Schedule** — run automatic schedule generation and make manual adjustments
